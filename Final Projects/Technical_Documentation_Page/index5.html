GIT

Git is a distributed version control system designed to track changes in source code during software development.

It allows multiple developers to collaborate on projects by providing mechanisms for managing, branching, merging, and tracking changes to codebase files. Git maintains a complete history of changes, enabling developers to revert to previous versions, analyze code evolution, and collaborate seamlessly across teams.

Getting Started with Git
How to Install Git:

1. Browse to the official Git website: https://git-scm.com/downloads

2. Click the download link for Windows (or depending to your OS) and allow the download to complete.

3. Browse to the download location (or use the download shortcut in your browser). Double-click the file to extract and launch the installer.

4. A Window will pop up, keep on clicking "Yes" or "Next" , note that everything is on default such as the Install Location,Text Editor,Component Selection, etc... Adjust to your own preferences

5.Once the installation is complete, tick the boxes to view the Release Notes or Launch Git Bash, then click Finish.

6. To check if Git has been installed successfuly, open your command prompt and type the command "GIT" if commands pop up you have successfuly installed GIT on your computer.

Basic Git Commands
1.git init:

Initializes a new Git repository in the current directory.

2.git clone [url]:

Clones an existing Git repository from a remote server to your local machine.

3.git add [file]:

Adds changes in a specified file to the staging area, preparing them to be committed.

4.git commit -m "[message]":

Records changes staged in the current branch with a descriptive message.

5.git status:

Displays the current status of the repository, including changes to be committed and untracked files.

6.git branch:

Lists all local branches in the repository.

7.git checkout [branch-name]:

Switches to the specified branch.

8.git merge [branch-name]:

Merges changes from the specified branch into the current branch.

9.git pull:

Fetches changes from the remote repository and merges them into the current branch.

10.git push:

Pushes local commits to the remote repository.

11.git log:

Displays a list of commits in the current branch, along with their metadata.

12.git remote -v:

Lists all remote repositories associated with the current repository, along with their URLs.

Collaborating with Git
1. Cloning remote repositories: git clone https://github.com/username/MyProject.git 2. Pushing and pulling changes: git add . git commit -m "Added new feature" git push origin master

If another developer has pushed changes to the remote repository, and you want to incorporate those changes into your local copy, you would use:

git pull origin master 3.Fetching and merging changes:

Suppose you're working on a feature branch called feature/new-feature and want to merge changes from the master branch into your feature branch:

git checkout feature/new-feature git fetch origin git merge origin/master
Git Tips and Tricks
1.Interactive Staging with git add -p:

When working on multiple changes within a file, you can interactively stage specific changes using the git add -p command. This allows you to review each change chunk by chunk and selectively stage them for commit.

git add -p 2.eflog for Recovery:

If you accidentally lose commits or changes, Git's reflog can be a lifesaver. The reflog keeps track of all commits and changes made in the repository, even those that have been reset or removed. You can use git reflog to view a log of all actions and find the commit you want to recover.

git reflog git checkout HEAD@{1} # Recover lost commit 3.quash Commits with git rebase -i:

You can combine multiple commits into a single, more coherent commit using interactive rebase (git rebase -i). This is particularly useful for cleaning up your commit history before merging changes into a main branch or creating pull requests.

git rebase -i HEAD~3 # Combine last 3 commits 4.Alias for Frequent Commands:

Create aliases for frequently used Git commands to save time and keystrokes. You can define aliases in your Git configuration file (~/.gitconfig) to create shortcuts for longer commands.

git config --global alias.co checkout git config --global alias.br branch git config --global alias.ci commit
WSL

WSL, or Windows Subsystem for Linux, is a feature of Windows that enables you to run Linux applications natively on your Windows machine. It provides seamless integration between Windows and Linux environments, allowing developers to leverage Linux tools and utilities directly within Windows without the need for virtual machines or dual-boot setups.

Installing WSL
Open your command prompt or PowerShell

Right Click and select "Run as administrator", enter the wsl --install command, then restart your Desktop/Laptop.

wsl --install
Ubtunu

Debian-based distributions provide a solid foundation for a wide range of computing tasks, from everyday desktop usage to server hosting, development, and specialized applications like ethical hacking or digital forensics. They offer a balance of stability, usability, and flexibility that makes them a popular choice for many users and organizations.

And one the most famous Debian-based distributions is Ubuntu

Installing Ubuntu

Open Command Prompt as an administrator:

Run the following command to download and install the Ubuntu distribution package:

wsl --install -d Ubuntu

Wait for the installation process to complete. This may take some time as it downloads the Ubuntu image and sets up the WSL environment.

Once the installation is finished, you can launch Ubuntu by typing ubuntu in Command Prompt and pressing Enter.

ubuntu
Sudo
Sudo

'sudo' is a command-line utility found in Unix and Unix-like operating systems

The primary purpose of 'sudo' is to enable users to perform tasks that require elevated privileges, such as installing or removing software, modifying system configurations, or accessing files and directories that are restricted to the root user.

Using 'sudo' Command

To run commands with administrative privileges, use the sudo command followed by the command you want to execute.

Example:

sudo apt update

This command updates the package lists for Ubuntu.
